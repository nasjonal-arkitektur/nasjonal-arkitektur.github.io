"Tjenester kan abonnere på ""Hendelser"" som grunnlag for videre agering.Tilby tjenester å spørre etter hendelser.Endringer som medfører en hendelse kan publiseres slik at andre tjenester kan agere på denne hendelsen.En hendelsesløsning som kan være konsument mot andre hendelsesløsninger.Dataminimering og tilgangsstyring av hendelser kan forvaltes og realiseres.Granulering av hendelser er mulig, og er basert på brukernes behov eller andre rammebetingelser (f.eks. om en hendelse inkluderer data om hendelsen i tillegg til selve hendelsen, eller ikke -> behov for eOppslag).Hendelser forvaltes med ulik krav og behov for levetid og tilbakerulling."
